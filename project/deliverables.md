# Cross-Domain Deliverables — OAuth2 Social Login Feature

## .pm/ (Product Management)
- research/ — Market research, competitor analysis, user personas
- gaps/ — Gap analysis scores and opportunities
- prds/ — Product Requirements Document (OAuth2 feature spec)
- experiments/ — Validation experiments, A/B test plans
- metrics/ — North star metrics, OKRs, success criteria
- stakeholder/ — Status reports, decision logs, alignment docs

## .pe/ (Principal Engineering)
- strategy/ — Technical feasibility, engineering strategy
- architecture/ — System design docs, architectural decisions
- decisions/ — Decision records, trade-offs

## .ux/ (UI/UX Design)
- design-systems/ — Design tokens, component hierarchy
- components/ — Sign-in component specs, OAuth flow diagrams
- interactions/ — Login modal micro-interactions, loading states
- accessibility/ — WCAG 2.2 audit (expected loop point)
- responsive/ — Mobile sign-in experience

## .sde/ (Senior SDE)
- requirements/ — Technical requirements, traceability
- designs/ — System design, OAuth2 flow architecture
- architecture/ — SOLID analysis, clean architecture review
- tests/ — TDD test strategy, unit/integration test specs
- code/ — Clean code assessment
- reviews/ — Code review findings (expected loop point: HIGH severity)
- estimates/ — Effort estimation with risk buffers

## .qae/ (Quality Assurance)
- strategies/ — Comprehensive test strategy
- plans/ — Detailed test plan with entry/exit criteria
- api/ — OAuth provider contract tests
- automation/ — Test automation framework
- exploratory/ — Session-based testing results
- defects/ — Defect reports, root cause analysis (expected loop point: HIGH severity defect)
- metrics/ — Test coverage, release readiness scorecard
